### 🏛️ 3. AWS 3-Tier Architecture — High Availability & Fault Tolerant

> *Production-grade AWS infra across 2 AZs — 99.9% uptime target, Multi-AZ RDS, dynamic Auto Scaling*

[![Repo](https://img.shields.io/badge/View_Repo-181717?style=for-the-badge&logo=github)](https://github.com/harshal019/AWS-3-Tier-Architecture)
![AWS VPC](https://img.shields.io/badge/VPC-FF9900?style=flat-square&logo=amazonaws&logoColor=white)
![RDS](https://img.shields.io/badge/RDS_MultiAZ-527FFF?style=flat-square&logo=amazonrds&logoColor=white)
![ALB](https://img.shields.io/badge/ALB-FF9900?style=flat-square&logo=amazonaws&logoColor=white)
![Auto Scaling](https://img.shields.io/badge/Auto_Scaling-FF9900?style=flat-square&logo=amazonaws&logoColor=white)
![IAM](https://img.shields.io/badge/IAM-DD344C?style=flat-square&logo=amazonaws&logoColor=white)

**Architecture:**
```
Internet ──► Route53 ──► External ALB
                              │
                     ┌────────┴────────┐
               AZ-1 (Public)    AZ-2 (Public)    ← Web Tier (Nginx)
                              │
                     Internal ALB
                              │
                     ┌────────┴────────┐
               AZ-1 (Private)   AZ-2 (Private)   ← App Tier (EC2 + ASG)
                              │
                     RDS MySQL Multi-AZ            ← Database Tier
```

- 🌐 Designed **3-tier architecture** separating web, app, and database layers across **2 Availability Zones**
- 🗄️ **RDS MySQL Multi-AZ** failover — database downtime under **60 seconds**
- 📊 **Auto Scaling Groups** (min 2 / max 5 EC2) — zero manual intervention during traffic spikes
- 🔐 End-to-end security: IAM roles · Security Groups · ACM SSL/TLS · NAT Gateway for private subnets
- 🏗️ Custom VPC with **4 subnets** (2 public, 2 private), Internet Gateway, Nginx reverse proxy
